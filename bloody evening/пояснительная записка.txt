    Пояснительная записка к программе "Кровавый Закат"
Программа "Кровавый Закат" представляет собой текстовую игру с визуальным сопровождением, разработанную с использованием библиотеки Pygame.
Игра построена на основе сценариев, где игроку предоставляется выбор, влияющий на развитие сюжета.
В игре используются изображения для фона, текстовые описания и музыкальное сопровождение.

    Основные компоненты программы:
1. Инициализация Pygame и настройки экрана:
   - Программа начинается с инициализации Pygame и создания окна с разрешением 1920x1080 пикселей. Название окна — "Кровавый Закат".
   - Определены основные цвета (белый, черный, красный) и полупрозрачный черный для фона текста.

2. Загрузка ресурсов:
   - Загружены фоновые изображения для каждой сцены, которые хранятся в словаре `backgrounds`. Каждое изображение масштабируется под размер экрана.
   - Загружен шрифт "Courier New" размером 30 пикселей для отображения текста.

3. Сцены и сюжет:
   - Сцены игры описаны в словаре `scenes`. Каждая сцена содержит последовательность текстовых сообщений, фоновое изображение и варианты выбора для игрока.
   - Каждая сцена имеет уникальный ключ (например, "intro", "open_case", "leave_case" и т.д.), который используется для переключения между сценами.
   - В каждой сцене есть последовательность текстовых сообщений, которые выводятся на экран с эффектом "печатающегося текста".
   После завершения текста игроку предлагается сделать выбор, который определяет следующую сцену.

4. Глобальные переменные:
   - `current_scene` — текущая сцена, которая отображается на экране.
   - `sequence_index` — индекс текущего текстового сообщения в последовательности сцены.
   - `typing_text`, `typing_index`, `last_typing_time`, `typing_speed` — переменные для управления анимацией текста, который "печатается" на экране.

5. Музыкальное сопровождение:
   - Загружена музыкальная композиция "music.mp3", которая воспроизводится в фоновом режиме бесконечно.
   - Реализованы функции `increase_volume` и `decrease_volume` для управления громкостью музыки.

6. Функция `render_scene`:
   - Отображает текущую сцену на экране. Функция принимает ключ сцены и индекс текущего текстового сообщения.
   - Текст выводится на экран с эффектом "печатающегося текста".
   Если текст еще не напечатан полностью, функция добавляет по одному символу с заданной скоростью typing_speed.
   - Текст отображается на полупрозрачном черном фоне для лучшей читаемости.

7. Функция `render_choices`:
   - Отображает варианты выбора для игрока после завершения текстовой последовательности сцены.
   - Варианты выводаются на экран в виде пронумерованного списка. Игрок может выбрать один из вариантов, нажав соответствующую цифру на клавиатуре.

8. Основной цикл игры `main`:
   - В бесконечном цикле программа проверяет, завершена ли текущая текстовая последовательность сцены. Если последовательность не завершена, вызывается функция `render_scene`. Если последовательность завершена, вызывается функция `render_choices`.
   - Программа обрабатывает события клавиатуры:
   - Пробел: Переход к следующему текстовому сообщению в текущей сцене.
   - Цифры 1, 2, 3: Выбор одного из вариантов, предложенных в сцене. В зависимости от выбора игрока, программа переключается на следующую сцену.
   - **Клавиши "+" и "-": Увеличение и уменьшение громкости музыки.

9. Переключение сцен:
   - Переключение между сценами происходит на основе выбора игрока. Каждая сцена содержит список возможных переходов `next`, которые соответствуют вариантам выбора.
   - При переходе на новую сцену сбрасываются переменные для анимации текста `typing_text` и `typing_index`, чтобы текст новой сцены начинал печататься с начала.

    Пример работы программы:
1. Игра начинается со сцены "intro", где игроку показывается текстовая последовательность с описанием сюжета. Текст выводится на экран с эффектом "печатающегося текста".
2. После завершения текста игроку предлагается выбор из трех вариантов:
   - Открыть кейс, который дал Блейк.
   - Оставить кейс закрытым и уйти.
   - Попытаться узнать больше о деле Блейка.
3. В зависимости от выбора игрока (нажатия клавиши 1, 2 или 3), программа переключается на соответствующую сцену (например, "open_case", "leave_case" или "search_blake").
4. В каждой сцене процесс повторяется: текстовая последовательность, выбор игрока, переход на следующую сцену.

    Управление громкостью:
- Игрок может увеличивать или уменьшать громкость музыки с помощью клавиш "+" и "-".

    Заключение:
Программа "Кровавый Закат" представляет собой интерактивную текстовую игру с визуальным сопровождением.
Основной акцент сделан на сюжет и выбор игрока, который влияет на развитие событий. Использование эффекта "печатающегося текста" и музыкального сопровождения создает атмосферу и вовлекает игрока в процесс.
Программа легко расширяема — можно добавлять новые сцены, варианты выбора и фоновые изображения для создания более сложного и увлекательного сюжета.
